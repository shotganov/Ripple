# Ripple

Проект является курсовой работой на курсе **«Т-Академия Фронтенда»**.

Автор: **Шаганов Егор Юрьевич**

## О проекте

**Ripple** — полнофункциональная социальная сеть для коротких публикаций и общения в реальном времени.

Приложение включает ленту постов, систему подписок, комментарии, уведомления, личные чаты, поиск пользователей, административную панель и инструменты мониторинга. Основной акцент проекта сделан на современной frontend/backend архитектуре, realtime-взаимодействии и наблюдаемости приложения.

## Задача проекта

Проект решает задачу создания лёгкой и минималистичной платформы для общения и публикации коротких текстовых сообщений.

Пользователи могут:

- публиковать посты;
- взаимодействовать с другими пользователями;
- читать персонализированную ленту;
- обсуждать публикации;
- общаться в личных сообщениях в реальном времени.

## Целевая аудитория

Пользователи, предпочитающие минималистичный формат общения: короткие публикации, тематические сообщества и персонализированную ленту без сложных алгоритмических рекомендаций.

## Возможности приложения

### Пользовательские возможности

- Регистрация и авторизация пользователей.
- JWT-аутентификация.
- Лента постов с бесконечной прокруткой.
- Создание постов с текстом и изображениями.
- Счётчик символов и эмодзи-пикер.
- Лайки и комментарии.
- Подписки на пользователей.
- Поиск пользователей и постов.
- Личные чаты в реальном времени через WebSocket.
- Система уведомлений.

### Профиль пользователя

- Редактирование профиля.
- Загрузка аватара и обложки.
- Изменение имени и биографии.

### Администрирование и модерация

- Жалобы на посты и комментарии.
- Система модерации контента.
- Удаление постов и комментариев.
- Панель администратора.
- Просмотр системных метрик и статистики.

### Мониторинг и observability

- Мониторинг ошибок через Sentry.
- Структурированное логирование через pino.
- Метрики HTTP- и WebSocket-активности через Prometheus.

## Архитектурные особенности

- Frontend построен по методологии Feature-Sliced Design (FSD).
- Backend реализован на NestJS с модульной архитектурой.
- Realtime-взаимодействие через Socket.IO.
- Prisma ORM + PostgreSQL.
- Docker-конфигурация для локального развёртывания.
- Role-based access control для административных функций.

## Запуск проекта

Проект — монорепо из двух независимых пакетов: `frontend` и `backend`. Команды запускайте из соответствующей директории.

### Вариант А — Docker (рекомендуется для бэкенда)

Требуется только Docker Desktop. PostgreSQL поднимается автоматически.

```bash
cd final-project-2-semester
docker compose up --build
```

При первом запуске автоматически применяются миграции и заполняются тестовые данные.

API будет доступен на `http://localhost:3000`.

Остановить и удалить контейнеры:

```bash
docker compose down
```

Удалить вместе с данными БД:

```bash
docker compose down -v
```

---

### Вариант Б — локальный запуск

### Требования

- Node.js ≥ 20
- PostgreSQL ≥ 14
- npm

### 1. Backend

```bash
cd final-project-2-semester/backend
npm install
```

Создайте базу данных в PostgreSQL:

```sql
CREATE DATABASE ripple;
```

Создайте файл `.env` в папке `backend/`:

```env
DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/ripple"
JWT_SECRET="any-long-secret"
```

Первичная настройка БД (генерация Prisma-клиента + миграции + сидинг тестовых данных):

```bash
npm run db:setup
```

Если БД уже была — пересоздать с нуля:

```bash
npm run db:refresh
```

Запуск в режиме разработки:

```bash
npm run start:dev
```

API поднимется на `http://localhost:3000`.

### 2. Frontend

```bash
cd final-project-2-semester/frontend
npm install
npm run dev
```

Приложение откроется на `http://localhost:5173`.
